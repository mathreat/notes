- 泛型

  泛型是java语言的一种编程特性，在编译时定义和限制类、方法、接口所操作的数据类型

  在 java 中泛型只是一个占位符，必须在传递类型后才能使用。类在实例化时才能真正的传递类型参数，由于静态方法的加载先于类的实例化，也就是说类中的泛型还没有传递真正的类型参数，静态的方法的加载就已经完成了，所以静态泛型方法是没有办法使用类上声明的泛型的。只能使用自己声明的 `<E>`

- 接口

  一种特殊的类型，可以理解为特殊的抽象类。例如public interface generator() ，与类是平级的

  定义方法规范，但没有实现。规范：接口定义了一组方法和常量，它规定了实现接口的类应具备的行为。这有助于在不同的类之间建立一种约定，让这些类都遵循相同的规范。对应的是重写。

  例子：

```java
public interface Flyable {
    void fly();
}
```

任何具有飞行能力的类（如鸟类、飞机等）都可以实现这个接口：

```java
public class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("鸟儿在飞翔");
    }
}

public class Airplane implements Flyable {
    @Override
    public void fly() {
        System.out.println("飞机在飞行");
    }
}
```

- I/O 流为什么要分为字节流和字符流呢?

  在Java中，I/O流被分为字节流和字符流的主要原因是为了处理不同类型的数据和满足不同的应用场景。字节流适用于处理**二进制数据**，而字符流适用于处理**文本数据**。在使用时，可以根据实际需求选择合适的流类型。例如，如果需要读写文本文件，可以使用字符流，因为它可以正确处理字符编码；而如果需要读写图像文件，则应使用字节流，以保证数据的原始性。

  

- 语法糖

  是一种特殊语法，为了方便程序员开发程序。例如`for-each`。JVM 其实并不能识别语法糖，Java 语法糖要想被正确执行，需要先通过编译器进行解糖，也就是在程序编译阶段将其转换成 JVM 认识的基本语法。

- 接口可以被类继承吗？

  在Java中，`Collection`接口和`List`接口之间存在继承关系。`Collection`接口是Java集合框架中所有集合类的根接口，它定义了一组通用的操作，如添加元素、删除元素、遍历元素等。`Collection`接口有多个子接口，例如`List`、`Set`、`Queue`等，这些子接口分别对应不同类型的集合。

  `List`接口是`Collection`接口的一个子接口，它表示一个有序的、可重复的元素集合。`List`接口继承了`Collection`接口的所有方法，并在此基础上添加了一些针对列表操作的特有方法，如根据索引获取元素、根据索引添加元素、根据索引移除元素等。

  总之，在Java中，`Collection`接口是集合类的根接口，而`List`接口是`Collection`接口的子接口，表示一种有序的、可重复的元素集合。`List`接口继承了`Collection`接口的通用操作，并添加了一些列表操作的特有方法。常用的`List`接口实现类有`ArrayList`、`LinkedList`等
